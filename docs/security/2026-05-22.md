## Security Findings

### HIGH Severity

#### H1: No Multi-Tenant Org Isolation
**Location:** `internal/org/`, `internal/auth/rbac/`  
**Finding:** The RBAC system has roles, permissions, and policies — but no `OrgID` or `TenantID` field. Users, roles, and permissions are global. In a multi-tenant deployment, any authenticated user can potentially access any org's resources.  
**Impact:** Cross-tenant data leakage in multi-org deployments.  
**Recommendation:** Add `OrgID` to `User`, `Role`, `Policy`, and all permission checks. Scoping `CheckAccess` to the user's org should be mandatory.

#### H2: No Memory Isolation Between Agents
**Location:** `internal/memory/`  
**Finding:** The `Memory` struct has `Agent` and `Session` fields but no org or agent-group scope. `Store.Search()` returns results across all agents. Any agent with access to the store can query any other agent's memories.  
**Impact:** Sensitive context (user data, API keys in memory) can leak between agents.  
**Recommendation:** Add `OrgID` to `Memory`, scope `Store` queries by org and agent. Enforce at the query level, not just API level.

#### H3: No Message Authentication (Agent Impersonation)
**Location:** `internal/comm/`  
**Finding:** `Message.From` is a plain string with no cryptographic verification. Any agent can send a message claiming to be any other agent. No HMAC, no signing, no token-based sender verification.  
**Impact:** Agent impersonation — a malicious/compromised agent could forge messages from the CEO agent or a compliance agent, potentially triggering unauthorized actions.  
**Recommendation:** Add per-agent signing keys. Include an HMAC or Ed25519 signature on every message. Verify on receipt.

#### H4: Cost Budget Checks Are Advisory, Not Enforced
**Location:** `internal/cost/tracker.go`  
**Finding:** `CheckBudget()` returns a status struct but does NOT block recording. `Record()` always succeeds regardless of budget status. `RecordUnchecked()` exists as an explicit bypass. The subagent spawner checks budget before parallel execution but nothing prevents a single agent from exceeding limits through repeated calls.  
**Impact:** Agents can exceed cost budgets with no hard enforcement. Budget is a suggestion, not a wall.  
**Recommendation:** Make `Record()` return an error when over budget. Remove or gate `RecordUnchecked()` behind admin approval. Add a hard circuit-breaker that rejects all LLM calls when a budget is exceeded.

### MEDIUM Severity

#### M1: Docker Sandbox Missing Hardening Flags
**Location:** `internal/suna/sandbox.go`  
**Finding:** Container creation uses CPU/memory limits and bridge networking, but omits `--security-opt`, `--cap-drop=ALL`, `--read-only`, and seccomp profiles.  
**Impact:** Containers have default Linux capabilities, increasing escape surface.  
**Recommendation:** Add `--cap-drop=ALL`, `--security-opt=no-new-privileges`, `--read-only` with tmpfs for writable paths, and a custom seccomp profile.

#### M2: Feedback Routing Depends on Ownership Registration
**Location:** `internal/feedback/feedback.go`  
**Finding:** `Route()` returns an error if no ownership mapping exists for a signal source. Signals from unregistered sources are silently stored but never routed.  
**Impact:** Production signals from new or misconfigured sources can be lost — the feedback loop breaks silently.  
**Recommendation:** Add a default/catch-all route. Alert when signals have no owner. Auto-register new sources with a default division.

#### M3: Quality Gate Unknown Criteria Silently Skipped
**Location:** `internal/qualitygate/qualitygate.go`  
**Finding:** Unknown `GateCriterion` types silently result in `StatusSkipped` with no warning or logging. A misconfigured gate (typo in criterion name) would pass without actually checking anything.  
**Impact:** Quality gates can be accidentally bypassed through configuration errors.  
**Recommendation:** Return `StatusFailed` for unknown criteria. Log a warning. Require explicit `StatusSkipped` to be opt-in.

---

### LOW Severity

#### L1: `isLikelySecret` Pattern Matching Is Fragile
**Location:** `internal/review/review.go`  
**Finding:** Pattern matching for secret detection uses substring matching and value heuristics. Can miss secrets in non-standard formats (base64-encoded, split across lines) and false-positive on documentation strings.  
**Recommendation:** Integrate with a dedicated secret scanning library (e.g., gitleaks) for production use.

#### L2: RBAC Admin Bypass Has No Audit Trail
**Location:** `internal/auth/rbac/rbac.go`  
**Finding:** `CheckAccess` returns `"admin bypass"` as the reason but doesn't log the bypass event to the audit log.  
**Recommendation:** Emit an audit event for every admin bypass.

#### L3: Task State Mutex Not Applied to All Paths
**Location:** `internal/subagent/subagent.go`  
**Finding:** While `GetState()`/`SetState()` were added, direct `task.State` reads remain in test assertions and some internal code paths (persistence, JSON serialization). These are safe in single-goroutine contexts but fragile for future changes.  
**Recommendation:** Audit all direct `task.State` access and replace with accessor methods.
